How this dashboard works

This site has no traditional backend, so this dashboard is a control panel over three real, working pieces rather than one unified system. Here's what each tab actually does:

🔔 Push Notifications
Opens your OneSignal dashboard (the reliable way to send) and offers an experimental quick-send tool.
🛠️ Technician Directory
Add/edit/remove technician profiles here, then copy the generated code into technicians.html.
📋 Clients & Applications
Every form submission is emailed to you AND (once set up) logged to a Google Sheet you can browse here.

Send a push notification

The reliable way to send is your OneSignal dashboard — it always works, has message history, scheduling, and audience filtering built in.

Open OneSignal Dashboard →

Tip: under Audience, filter by tag user_type = technician for job leads, or user_type = customer for bookings/promos — every subscriber is already tagged automatically by the site.

Quick Send (experimental)

Sends directly from this page via OneSignal's API. This may be blocked by your browser (OneSignal's API isn't guaranteed to allow direct browser calls) — if it fails, use the dashboard above instead. Your API key is never saved; it only exists in this browser tab while you use it.

Get your REST API Key from OneSignal → Settings → Keys & IDs. Treat it like a password — anyone with it can send notifications to all your subscribers.

Add / edit technician profiles

This builds the exact code block used by technicians.html. Add or remove profiles below, then copy the generated code and paste it in as instructed — that one paste fully updates the live directory.

Never enter a technician's phone number or email here — this list becomes public-facing code. Only name, trade, experience, location, bio, and photo URLs belong here.

Current directory list

This list is saved in this browser only. The live site isn't updated until you paste the copied code into the TECHNICIANS array in technicians.html and re-upload it.

Set up the Google Sheet first

config.js doesn't have a Sheets endpoint configured yet, so form submissions are only arriving by email right now. Follow the setup instructions at the top of google-apps-script.gs (about 5 minutes), then paste the resulting URL into config.js and re-upload it.

Once that's done, this panel will let you jump straight to your Bookings, Technician Applications, and Contact Requests tabs.

Your records

All three live as tabs inside one spreadsheet. Paste your sheet's URL once below to save quick links — every submission from the site lands here automatically, in addition to your email.
